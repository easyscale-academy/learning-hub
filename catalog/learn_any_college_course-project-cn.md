# AI-Powered College Course Mastery

## Overview

**传统学习的痛点：**

- 教材厚得像砖头，专业术语满天飞，不知道从哪里下手
- 资料四散在各处：网盘、邮箱、聊天记录、纸质笔记
- AI 聊天工具顶多对话，不能真正操作文件、整理笔记
- 学完一学期，笔记乱七八糟，考完就忘，知识没有积累
- 和教授、TA 沟通时，脑子没想清楚就开口，聊完发现重点没问到

**我们的解决方案：**

一套完整的 AI 辅助学习系统，让 AI 成为你的私人助教——它知道你在学什么课、读到哪一章、哪里不懂。它能帮你收集整理资料、拆解复杂概念、生成学习笔记、准备沟通脚本，甚至帮你搜索海量资料中的任何内容。

| 传统学习方式 | 我们的方案 |
|------------|-----------|
| 资料散落各处，找不到 | 统一的本地知识库，AI 智能搜索 |
| 教材太厚，读不完 | AI 按章节拆分，按需阅读 |
| 概念太难，学不懂 | AI 用通俗语言讲解，举例子，出题检验 |
| 笔记乱七八糟 | 结构化存储，版本控制，永不丢失 |
| 和老师沟通没准备 | AI 帮你理清思路，生成沟通脚本 |
| AI 不知道你在学什么 | 课程信息持续生效，AI 始终了解上下文 |

## 模块 1: 概述与框架

### 问题

学习方式零散：今天在这个 app 做笔记，明天在那个网站看视频，后天在另一个地方交作业。信息四处分散，无法形成完整的知识地图。

### 解决方案

**一套体系、一整套工具、一套行为模式。**

- **一套体系**：基于 Git 的本地知识库，每门课是一个分支，所有资料结构化存储
- **一整套工具**：针对不同任务优化的 AI 助手，一个命令完成一类任务
- **一套行为模式**：主动收集、持续整理、深度思考、知识积累

### 核心价值

这不只是工具的堆砌，而是一种全新的学习方式：
- 从「被动接收」变成「主动构建」
- 从「考完就忘」变成「知识资产」
- 从「临时抱佛脚」变成「持续积累」

## 模块 2: 网页转 Markdown

### 问题

教授给你课程网站链接——syllabus、作业要求、阅读材料。但原始 HTML 充满样式代码和广告，直接喂给 AI 会浪费大量 token，回答质量也差。

### 解决方案

用浏览器开发者工具选择特定内容区域，通过在线工具转换成干净的 Markdown，再让 AI 进一步清理。整个过程 10 秒完成，无需任何代码。

### 输出示例

```markdown
# CS 261 - Data Structures

## Course Information
- Instructor: Dr. Rob Hess
- Office Hours: Mon/Wed 2-3pm

## Grading
- Assignments: 40%
- Midterm: 25%
- Final: 35%
```

## 模块 3: PDF 拆分与索引

### 问题

大型 PDF 教材（几百页的教科书）对 AI 来说是噩梦：
- 读取慢，token 消耗高
- 找信息困难，AI 容易迷失重点
- 每次都要重新处理整本书

### 解决方案

AI 自动分析 PDF 的章节结构，智能拆分成小文件，生成目录索引。一次设置，整学期受益——需要哪章读哪章。

### 输出示例

```
教材.pdf (500页)
    ↓ AI 分析与拆分
├── 教材-index.md      ← 目录索引
├── chapter-01.pdf     ← 第1章 (15页)
├── chapter-02.pdf     ← 第2章 (22页)
├── chapter-03.pdf     ← 第3章 (18页)
└── ...
```

## 模块 4: 教材翻译

### 问题

用 Google 翻译看英文教材，翻译出来的中文你也看不懂——因为概念本身就很抽象，逐字翻译解决不了理解问题。

### 解决方案

AI 不是逐字翻译，而是用你的母语**重新讲解概念**。技术术语保留英文（方便日后学术交流），但解释用中文，让你真正理解。

### 个性化体现

系统会根据你设置的第二语言自动生成双语内容。设置一次，全局生效。

## 模块 5: 目录结构

### 问题

没有统一的组织方式，资料越多越乱。找个文件要翻半天，AI 也不知道去哪里找信息。

### 解决方案

标准化的目录结构：

```
你的课程/
├── references/          ← 教材、课件、参考资料
│   └── schedule/        ← 按周组织的学习材料
├── assignments/         ← 作业和项目
├── notes/               ← 学习笔记
├── communication/       ← 沟通记录
└── COURSE_INFO.md       ← 课程信息速查表
```

每个目录都有 `INDEX.md` 索引文件，AI 只需读取索引就能了解整个目录结构，大幅提升搜索效率。

## 模块 6: 资料导入

### 问题

开学第一周，教授上传了 20 个 PDF、5 个视频链接、3 个网站。你不知道先看哪个，也不知道怎么保存才方便以后找。

### 解决方案

AI 像一个智能爬虫，自动：
- 访问课程网站，识别所有资料链接
- 下载 PDF、slides、文档
- 按类型和时间分类整理
- 生成课程信息摘要

### 输出示例

```markdown
# COURSE_INFO.md

## 基本信息
- 课程: CS 261 - Data Structures
- 教授: Dr. Rob Hess
- 学期: Fall 2020

## 考试安排
- 期中: Week 5, 时长 2 小时
- 期末: Finals Week

## 联系方式
- Office Hours: Mon/Wed 2-3pm, Office 302
- Email: hessro@oregonstate.edu
```

## 模块 7: 自主学习

### 问题

传统学习的致命问题：**反馈太慢**。读了一章书，以为自己懂了，等到考试才发现不懂。想找人问又不好意思，就这么一直卡着。

### 解决方案

AI 作为你的「学习拍档」，实现**主动阅读 + AI 辅助理解 + 即时检验**的学习循环。

**学习循环：**

```
读原文 → 遇到不懂的 → 问 AI
    ↑                    ↓
    ← 理解了 ← AI 讲解/举例/出题
```

**核心能力：**
- 用通俗语言讲解复杂概念
- 生成聚焦的可执行示例
- 即时出题检验理解
- 把学习内容总结成笔记
- 追踪学习进度，记住你学到哪里

### 输出示例

```markdown
# Big-O Notation 学习笔记

日期：2026-03-15
来源：Chapter 2, pages 10-15

## 核心概念
Big-O 表示法描述算法性能如何随输入规模增长...

## 关键公式
- O(1): 常数时间
- O(n): 线性时间
- O(n²): 平方时间

## 易错点
- 忽略常数系数和低阶项
- 混淆最坏情况和平均情况
```

## 模块 8: 高效沟通

### 问题

去找教授，脑子没想清楚就开口。支支吾吾聊了半小时，出来后发现真正想问的问题根本没问到。

### 解决方案

把沟通变成有章可循的流程：

```
头脑风暴（和 AI 理清思路）
    ↓
生成脚本（问题清单 + 信息清单 + 话术参考）
    ↓
执行沟通（带着脚本去聊）
    ↓
记录内容（结构化记录 + 行动项）
    ↓
跟进行动（追踪完成情况）
```

**三种沟通场景：**
- Office Hour：准备问题清单，记录教授建议
- 电话：准备通话脚本，记录关键信息
- 邮件：撰写正式邮件，分析收到的回复

### 输出示例

```markdown
# Office Hour 准备脚本

## 背景
CS101 期末项目方向讨论

## 要问的问题
- [ ] 机器学习项目是否适合 CS101？
- [ ] 合理的项目范围是什么？
- [ ] 有往届学生的项目示例吗？

## 要告诉教授的
- [ ] 对推荐系统方向感兴趣
- [ ] 已完成所有课程作业
- [ ] 每周可投入 15-20 小时

## 参考话术
"Hi Professor Chen, I'm considering a machine learning project..."
```

## 模块 9: 搜索与索引

### 问题

资料越来越多——教材、讲义、作业、笔记、邮件、代码……期末复习时想找「哈希表」相关的内容，翻了半小时还没找全。

### 解决方案

**三层搜索架构**：

```
第一层：文件名匹配（最快，免费）
    ↓
第二层：内容搜索（快，精确匹配）
    ↓
第三层：AI 语义理解（智能，理解上下文）
```

先用传统搜索快速筛选，缩小范围到 5-10 个候选文件，再让 AI 深度分析。既快速又智能，还省 token。

**五种搜索方式：**
- 全局搜索：不知道资料在哪时使用
- 教材搜索：只搜索参考资料目录
- 作业搜索：只搜索作业和代码
- 笔记搜索：只搜索个人笔记
- 沟通搜索：只搜索邮件和会议记录

### 输出示例

```
🔍 搜索结果: 哈希表

[1] 📄 references/lectures/week-05-hashing.md
    └─ 匹配：文件名包含 "hashing"

[2] 💻 assignments/hw3-hashtable-impl.c
    └─ 匹配：内容包含 "hash table"

[3] 📝 notes/2026-03-05-hash-table-notes.md
    └─ 匹配：内容包含 "哈希表"

下一步：输入数字查看详情，或继续提问
```

## 为什么这套流程有效

### 传统方式的痛点

1. **信息碎片化**：资料散落在各处，找不到、管不好
2. **反馈延迟**：学了不知道懂没懂，等到考试才发现问题
3. **知识易逝**：考完就忘，无法积累
4. **沟通低效**：没准备、没记录、没跟进
5. **AI 割裂**：每次对话从零开始，AI 不了解你的上下文

### 我们的解决方案

1. **统一知识库**：所有资料结构化存储，一处管理、随处可访问
2. **即时反馈**：学一点验一点，AI 随时可问、随时出题
3. **知识资产化**：笔记版本控制，学习记录永久保存
4. **沟通系统化**：准备→执行→记录→跟进的闭环流程
5. **上下文持续**：AI 始终了解你的课程、进度、需求

### 核心优势

| 维度 | 传统学习 | 我们的方案 |
|-----|---------|-----------|
| 资料管理 | 散落各处 | 统一知识库 |
| 学习反馈 | 考试才知道 | 即时检验 |
| 知识保留 | 考完就忘 | 永久积累 |
| AI 体验 | 每次从零开始 | 持续上下文 |
| 沟通效率 | 想到哪说到哪 | 有章可循 |
| 搜索能力 | 手动翻找 | 智能检索 |

## 核心价值

**学习不是消费信息，而是生产知识。**

传统学习把学生当作信息的被动接收者——上课听讲、回家做题、考前突击。这套系统重新定义了学习的姿态：你是知识的主动构建者，AI 是你的工具和拍档。

**三个根本性的改变：**

1. **从碎片到体系**：散落的资料变成结构化的知识库，可搜索、可追溯、可积累
2. **从延迟到即时**：学习反馈从「考试才知道」变成「随时检验」，问题在萌芽时就被发现
3. **从孤立到连接**：AI 始终了解你的课程上下文，每次对话都是上一次的延续

AI 不是替代你思考，而是帮你思考得更好。它是放大器，不是拐杖。

最终，你收获的不只是某门课的成绩，而是一种可迁移的学习能力——无论未来学什么，你都有方法、有工具、有系统。
