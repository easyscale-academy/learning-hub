# Tech Interview Know-How Mastery

## Overview

**传统面试准备的痛点：**

你用过 Kafka、Spark、MongoDB——简历上写得清清楚楚。但当面试官问"你们为什么选 Kafka？"的时候，脑子突然一片空白。

不是因为你不会用这些技术。而是因为你从来没想过：**它为什么存在？在它之前人们怎么做？什么时候不该用它？**

这类问题叫 **Know-How 问题**。面试官问的几乎全是 Know-How，而大多数人准备的几乎全是 Know-What。

**我们的解决方案：**

一套系统化的 AI 辅助学习流程，帮你从"用过这个技术"升级到"真正理解这个技术"——在面试官发现你的盲区之前，先找到它们，然后补上。

| 传统方式 | 我们的方式 |
|---------|----------|
| 靠经验积累 Know-How | 用 AI 系统性挖掘盲区 |
| 看文档背概念 | AI 根据你的简历定制学习内容 |
| 面试时才发现说不清楚 | 模拟面试提前暴露问题 |
| 每次面试从零准备 | 积累个人知识库，越用越厚 |

## 模块 1: 盲区诊断

### 问题

"我用过 Kafka" 和 "我理解 Kafka" 之间有一道隐形的鸿沟。这道鸿沟平时看不见——直到面试官问了正确的问题。

大多数人直到面试现场才发现自己的盲区。那时候已经晚了。

### 解决方案

AI 分析你的简历和目标岗位，自动生成一批针对你背景的 Know-How 问题——这些正是面试官会问的问题。

你只需要诚实地给每个问题打标签：
- **空白**：完全答不上来，需要深入学习
- **模糊**：能说但说不清楚，需要厘清
- **扎实**：能自信回答，是面试优势

这就是你的盲区地图——面试前的精准学习路线图。

### 个性化体现

- 问题基于**你的简历**生成，不是泛泛的技术问答
- 结合**目标岗位要求**，聚焦最可能被问到的方向
- 可以针对特定技术深挖，获得更细分的问题

## 模块 2: 技术深挖

### 问题

你发现了盲区，但怎么补？

搜索引擎找到的资料要么太浅（告诉你怎么用，不告诉你为什么），要么太散（东一篇西一篇，拼不起来）。你需要的是一个懂你背景、能针对性讲解的技术导师。

### 解决方案

AI 扮演你的私人技术导师。你把问题和简历一起提供，它会：

1. **先问你几个问题**——搞清楚你卡在哪个层面
2. **生成定制综述**——根据你的背景、问题、期望深度量身定制
3. **支持无限追问**——每个追问都让理解更深一层

学完之后，技术不再是简历上的一个"点"，而是一条有来龙去脉的"线"。

### 个性化体现

- AI 看到你的项目经历，知道从什么角度切入
- 不是泛泛的科普，而是回答**你具体的问题**
- 追问过程中，你学会提出更尖锐的问题——这本身就是进步

### 输出示例

一篇完整的技术综述，覆盖：
- 这个技术为什么存在？解决什么问题？
- 在它之前人们怎么做？
- 它和竞品的本质区别是什么？
- 什么场景下不该用它？
- 关键的设计决策和 trade-off

## 模块 3: Know-How 模拟面试

### 问题

知识补了，笔记也攒了。但有一个问题：**你真的能在面试压力下把它说出来吗？**

知道和说出来是两回事。打字的时候你有时间组织语言，说话的时候你没有。

### 解决方案

AI 扮演真正的面试官——不是一问一答的假面试，而是会追问、会逼你把话说清楚的那种。

流程：
1. 提供简历 + JD + 一个 Know-How 问题
2. AI 像面试官一样问你，追问最多三轮
3. 你用**语音**回答，用**英语**回答
4. AI 给你反馈，并检查你的知识库

关键创新：AI 会告诉你这个问题**你是学过但忘了，还是压根没学过**。学过但忘了？回去复习。没学过？回去补课。这就是整个系统的闭环。

### 个性化体现

- 追问基于**你说的话**展开，不是预设脚本
- 反馈指出**你个人的问题**：是表达不清、还是知识缺口
- 知识库检查让你明确**下一步该做什么**

## 模块 4: 项目深挖模拟面试

### 问题

Know-How 考察的是你懂不懂一个技术。但面试还有另一关：**你能不能把你做的事情讲清楚？**

Project Deep Dive 面试中，面试官会挑简历上的一个项目，往深里挖：
- 当时是什么情况？你们要解决什么问题？
- **你**具体做了什么（不是团队，是你）？
- 你为什么做出那些决策？
- 遇到了什么困难？怎么解决的？

大多数人挂掉不是因为没做好——而是因为在压力下讲不清楚。

### 解决方案

AI 扮演面试官，从多个角度深挖你的项目：

- 背景和问题定义
- 你的个人贡献（"我做了什么"，不是"我们做了什么"）
- 决策背后的"为什么"
- 遇到的挑战和解决方案
- 可量化的结果和影响
- 反思和改进空间

5-8 轮追问后，你会收到结构化反馈：叙事清晰度、技术解释能力、自我认知深度。

### 个性化体现

- 问题基于**你简历上的真实项目**
- 追问根据**你的回答**动态调整
- 反馈告诉你**你的**叙事习惯问题：是啰嗦、是跳过了为什么、还是分不清个人和团队贡献

## 为什么这套流程有效

### 传统方式的痛点

1. **泛泛地学技术**：知道怎么用，不知道为什么。知识是孤立的点，面试时容易被追问到哑口无言。

2. **被动积累经验**：等着工作中慢慢学 Know-How。但面试不会等你有三年经验——它现在就要问你。

3. **面试时才发现问题**：第一次知道自己说不清楚，是在真正面试的时候。那时候 offer 已经丢了。

4. **每次从零准备**：没有系统性积累。上次面试准备的东西，下次面试还得重新来。

### 我们的解决方案

1. **主动挖掘盲区**：在面试官发现之前，先找到自己的知识缺口。

2. **定制化学习**：AI 根据你的简历和问题生成内容，不是泛泛的教程。

3. **压力测试表达**：在安全的环境里练习说话，在丢 offer 之前发现问题。

4. **知识库持续积累**：每一篇综述、每一次练习都存下来。知识库越用越厚，准备越来越快。

### 核心优势

- **针对你**：所有问题、综述、模拟都基于你的背景定制
- **闭环系统**：诊断 → 学习 → 检验 → 发现新缺口 → 继续学习
- **可量化进步**：盲区地图的变化可见，知识库的增长可见
- **终身资产**：知识库跟着你的职业发展，每次面试不是从零开始
