# AI 编程面试助手

## Overview

**传统刷题方式的痛点：**

- 读不懂题，搜资料质量参差不齐
- 看了题解还是不会做，没人给你解释
- 好不容易做出来，下周全忘了
- 在网页编辑器里调试，反馈慢、体验差
- 刷了几百道题，什么都没留下

**我们的解决方案：**

一套 AI 驱动的编程面试准备系统。AI 不只是回答问题，而是像一位有经验的导师，帮你系统化地理解、练习、记录。

| 维度 | 传统方式 | 我们的方式 |
|-----|---------|----------|
| 读题 | 自己看，看不懂上网搜 | AI 用你能懂的语言解释 |
| 思路 | 看题解，看不懂就卡住 | AI 把大问题拆成小问题 |
| 测试 | 平台给什么用什么 | 为你生成个性化测试 |
| 反馈 | 提交后等平台判题 | 本地即时运行测试 |
| 留存 | 做完就忘 | 完整文档保存在 GitHub |

---

- 基本的编程基础（Python 或 SQL）
- 安装好开发环境（mise、Python）

## 模块 01: 认知升级

### 问题

刷了几百道 LeetCode，面试还是紧张，代码还是写不出来。问题出在哪里？

### 解决方案

通过对比三个时代的学习方式（传统刷题 → 早期 AI → AI Agent），帮你理解：

- 传统平台的测试是为了**验证**，不是为了**教育**
- AI 的真正价值是把大问题拆小，把抽象概念具体化
- 学习的产出应该是你自己的知识库，而不是平台上的徽章

### 个性化体现

本模块是阅读材料，帮你建立正确的学习心态。

## 模块 02: 算法题实战

### 问题

面对一道 Hard 算法题，不知道从哪里下手。网上的题解看不懂，代码抄过来也不会改。

### 解决方案

一套六步工作流，用 AI 工具系统化攻克任何算法题：

```
理解题目 → 拆解思路 → 生成测试 → 写代码 → 随机验证 → 优化解法
```

**核心特点：**

- **读题**：AI 用简单语言解释，直到你真正理解
- **思路**：把大问题拆成小问题，每个小问题都能攻克
- **测试**：为你生成个性化测试，在本地即时反馈
- **代码**：从最小子问题开始，一步步往上堆
- **验证**：随机测试发现边界 bug
- **优化**：分析复杂度，探讨更优解法

### 个性化体现

- AI 根据你的水平调整讲解难度
- 测试用例根据你的理解程度简化或复杂化
- 每道题的解读文档、代码、测试都保存在 GitHub

### 输出示例

每道题完成后，你会得到：

```
problem/
  explain.md     # 题目解读
  hint.md        # 解题思路
  optimize.md    # 优化分析

learn_coding_interview/
  solution.py    # 你的代码

tests/
  test_solution.py        # 测试用例
  test_solution_random.py # 随机测试
```

## 模块 03: SQL 题实战

### 问题

SQL 面试越来越常见，但表结构看不懂，JOIN 写不对，本地又没法测试。

### 解决方案

同样的六步工作流，针对 SQL 做了调整：

- 你写的是 `.sql` 文件，不是 Python
- AI 在本地创建测试数据库，数据随时可查
- 查询结果用表格打印，一目了然

**SQL 专属优势：**

- 帮你理解表之间的关系
- 解释 JOIN、GROUP BY、窗口函数等概念
- 处理不同 SQL 方言的差异（MySQL vs SQLite）

### 个性化体现

- 表数据根据题目自动生成
- 可以看到查询中间结果，而不只是最终答案
- 本地测试通过后再提交平台，一次过率大幅提升

## 模块 04: 白板面试理论

### 问题

算法会做了，但上了白板就大脑空白。面试考的不只是算法，还有表达、互动、在压力下写出无 bug 代码的能力。

### 解决方案

一套完整的白板面试指南，覆盖：

- 面试官真正在评估什么（不只是算法）
- 白板面试的标准六步节奏
- Comment-Driven Development：先写注释再写代码
- 如何与面试官有效互动
- Bug-free coding 的四种训练方法

### 核心洞见

- 白板面试是真实工程工作的压缩模拟
- LeetCode 训练「会不会」，白板训练「能不能一次写对」
- 沉默的思考是看不见的思考——永远要出声

## 模块 05: 白板面试模拟

### 问题

知道方法论是一回事，真正做到是另一回事。需要一个安全的环境来练习「边想边说边写」。

### 解决方案

一位 AI 面试官，模拟真实的白板面试体验：

- **和面试官对话**：用文字写下你的思考，AI 给出反馈
- **卡住时获取提示**：AI 用苏格拉底式提问引导你，而不是给答案
- **面试结束获取点评**：详细的面试反馈报告

**关键设计：**

- AI 永远不会给你代码，答案要自己想
- 全程英文交流，模拟真实面试场景
- 你写代码，AI 只读不写

### 个性化体现

面试结束后，AI 生成一份详细的反馈报告：

- 问题解决能力评分（40%）
- 沟通表达能力评分（30%）
- 代码质量评分（20%）
- 技术深度评分（10%）
- 做得好的地方（继续保持）
- 需要改进的地方（具体怎么练）
- 推荐的下一步练习

### 输出示例

```
problem/
  talk.md     # 和面试官的对话记录
  review.md   # 面试反馈报告

learn_coding_interview/
  solution.py # 你的白板代码
```

## 为什么这套流程有效

### 传统方式的痛点

1. **碎片化学习**：题目、资料、代码分散在不同地方
2. **被动接受**：平台给什么就做什么，没有主动权
3. **没有反馈**：做完了不知道哪里好、哪里差
4. **没有留存**：刷完就忘，没有积累

### 我们的解决方案

1. **系统化流程**：每道题都走完整的六步，形成肌肉记忆
2. **AI 个性化辅导**：根据你的水平调整，不会太难也不会太简单
3. **即时反馈循环**：本地测试，秒级反馈，快速迭代
4. **完整知识库**：所有文档和代码保存在 GitHub，随时复习

### 核心优势

| 能力 | 如何训练 |
|-----|---------|
| 读懂题目 | AI 多角度解释，直到你真正理解 |
| 拆解问题 | AI 把大问题分解成可攻克的小问题 |
| 写出正确代码 | 从小问题开始，边写边测 |
| 一次写对 | Comment-Driven Development + 本地测试 |
| 面试表达 | 和 AI 面试官对练，获取反馈 |
| 知识留存 | 完整文档保存，随时复习 |

## 适用范围

### 适用于什么学科/场景

- **算法面试**：LeetCode、HackerRank 等平台的算法题
- **SQL 面试**：数据工程、后端开发、数据科学岗位的 SQL 题
- **白板面试**：需要边写边说的技术面试场景

### 适用于什么学习阶段

- **初学者**：AI 会用最简单的语言解释概念
- **有基础**：快速攻克题目，建立知识体系
- **准备面试**：模拟真实面试场景，训练元技能

### 适用于什么信息载体

- **截图**：直接截取 LeetCode 题目
- **文本**：复制粘贴题目描述
- **PDF**：保存的面试题
- **代码模板**：平台提供的函数签名

## 核心价值

> 学习的最终产出应该是你自己的知识库，而不是平台上的一个「已通过」徽章。

每道题做完，你带走的是：

- 一份你能看懂的题目解读
- 一份拆解清晰的解题思路
- 一套你能运行的测试代码
- 一份你自己写的解法
- 一份复杂度分析和优化建议
- （白板模式）一份详细的面试反馈报告

**这些都是你的，永远是你的。**
